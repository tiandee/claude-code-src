# 关于从 Claude Code 源码提炼 Agent Harness 架构设计模式的分析报告

近期，我们对 Anthropic 官方发行的智能编码助手 Claude Code（内部代号：Tengu）的核心源码进行了深度剖析。作为一款生产级别的代码智能体，其底层架构设计具备高度的前瞻性与工程完备性。为有效指导我们下一阶段 Agent Harness 框架的设计与研发，现将该源码中最具借鉴价值的七项核心设计模式总结报告如下：

## 一、采用 AsyncGenerator 驱动的流式主循环机制

该机制是系统内最核心的基础架构选择。其主循环引擎（Query Engine）采用原生异步生成器（AsyncGenerator）实现无限轮询。

**（一）天然支持流式输出**
引擎每产生一个 Token 或推进一个工具执行进度，即刻向上层渲染节点挂起（yield），彻底打破了传统框架“等待完整响应后分发”的阻塞式通信链路。

**（二）机制层面的请求背压支持**
凭借异步迭代器的特性，当消费端（如终端 UI 或客户端）处理性能出现瓶颈时，主循环会自动触发背压（Backpressure）机制暂停生成，有效保障了系统高并发下的内存安全。

## 二、推行“流式并行”的工具编排策略

不同于传统智能体在完整解析大语言模型（LLM）工具调用（tool_use）块后方执行工具的做法，该系统创新性地实现了“边接收数据流边触发工具”的编排模式。

**（一）状态机与工具提前启停**
内置流式执行器（StreamingToolExecutor）独立维护每个工具的状态机。一旦捕获到首个完整的 `tool_use` 块，系统立即校验并拉起执行进程，无需等待后续 Token。

**（二）基于输入属性的动态并发鉴权**
工具的并发能力由静态定义转变为动态判定。各个工具需暴露 `isConcurrencySafe` 接口，依据当前传入的参数动态声明自身的并发安全性。例如，`Bash` 工具在执行只读命令（如 `ls`）时允许并行，而执行写操作时强制串行。系统底层将基于此进行动态的任务分区编排。

## 三、建立四级递进式上下文压缩体系

长文本对话的上下文雪崩是业界共性难题。该架构建立了一套从轻到重、梯度防御的四级截断与压缩策略。

**（一）前置免费截断层（一至三级）**
1. **Snip Compact 裁剪策略**：直接在消息流顶端截断冗余历史消息列，零 Token 开销。
2. **Micro Compact 微缩策略**：以隐藏原始响应、仅保留调用标识（tool_use_id）的方式对工具返回体进行微缩。
3. **Context Collapse 折叠策略**：对验证完成的子任务执行链路进行全文折叠，替换为短效摘要。

**（二）深层大模型压缩机制（四级）**
当且仅当前三级均无法将上下文缩排至阈值内时，系统方才触发第四级 **Auto Compact**，调用 LLM 对对话结构化归纳。系统为该层设置了硬性熔断器（连续三次失败则终止重试），并默认预留 13K 缓冲窗口（Buffer Window）应对突发长文本。

## 四、确立三种维度的 Multi-Agent 协调协同模型

系统内规避了单体 Agent 容量不足的问题，基于隔离强度拆分了三项生命周期模型：

**（一）Fork 共享型子智能体**
通过构造与父节点完全一致的字节级 API 请求前缀，触发大语言模型的 Prompt Cache 缓存命中，实现父子节点零负担的上下文继承。

**（二）Worker 隔离型子智能体**
从全局资源中抽取创建全新的执行上下文。带有独立的终止控制器（AbortController）及全局缓存（FileStateCache），互不侵染。

**（三）Coordinator 编排型协调器**
在特定模式下启用纯编排能力。按照“需求调研（Research）→方案合成（Synthesis）→执行落地（Implementation）→检验验证（Verification）”四阶段流转子任务，并通过标准化 XML 文本交换结果。

## 五、实施支持权限冒泡的钩子拦截网

安全权限模型从单点拦截升级为具备穿透能力的立体网络。

**（一）多样化审批权限域**
从严格审查（plan）到全放行（bypassPermissions），构筑了共 6 种权限层级分类。

**（二）父子进程权限冒泡规则**
引入 `bubble`（冒泡）模式。当隔离执行的子智能体遭遇低权限壁垒时，可将权限请求对象化并抛转至父进程，交由统一的人类审批层判定。

**（三）错误容忍式的生命周期钩子**
拦截器网络被设计为“强容错”。权限拦截抛出的 `blocking error` 被视作工作流的一部分，大模型将以此为基础自主纠偏并推进任务，而非触发系统灾难性崩溃。

## 六、引入“特征门控（Feature Flag）”治理代码分支

**（一）编译时死代码清除**
架构核心逻辑中内嵌 `feature('KEY')` 标记，配合底层编译平台的打包机制，对未启用特性实施代码级物理剥离。

**（二）运行时云端分发**
全面接入 GrowthBook 等平台，通过 `tengu_*` 命名的云控开关，实现关键逻辑的线上灰度降级与 A/B 并行验证。

## 七、部署精细化的资源与成本探针

摈弃粗放的 Token 计费，实施精准到动作粒度的财务与追踪管理：
1. **维度穿透**：全量独立记录 `input_tokens`、`output_tokens` 甚至最新的 `cache_read` 和 `cache_creation` 指标。
2. **生命周期挂载**：每次触发上下文摘要重写或启动子 Agent，均开辟独立的计费账单，最终聚合至当前 Session 进行统一持久化，确保中断后复现时账目的强一致性。

---

**总结**：Claude Code 在处理主循环调度、动态并发、阶梯压缩以及权限冒泡等深水区难题上做出了极佳的设计示范。这些模式已被验证满足超大规模工业化要求，对我们自研 Agent Harness 的整体技术架构选型具有重要的战略及战术指导意义。请相关开发团队认真研读并在后续底座重构方案中充分借鉴。
